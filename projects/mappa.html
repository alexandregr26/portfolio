Project Information

This project was part of the University of Toronto's Engineering second-year design course ECE297, and it involved designing and completing a working application similar to Google Maps in a team, and communicating effectively with both technical supervisors and less technical clients.

.

Team & Strategy

Forming the team was the first stage of this project. In this stage, all group members became acquainted and agreed on the project goals, created a team charter, and established individual roles. The group prioritized a clear communication system, both personal and technical, to increase the individual effort of every member and the willingness to cooperate. Weekly meetings were held and Git, a version control software tool, was chosen to help manage the source code of the project and prevent concurrent work from conflicting.

Building the Infrastructure

The first part for this project was to take advantage of two APIS that allowed us to query a geographic information database for information such as street names and intersection locations. To accomplish this, functions and data structures were created and implemented to store and retrieve data.

#include "StreetsDatabaseAPI.h"

//Returns the street segments for the given intersection 
std::vector<int> find_street_segments_of_intersection(int intersection_id) {
    std::vector <int> streetSegmentsIDS;
    streetSegmentsIDS = IntersectionsOfMap[intersection_id];
    return streetSegmentsIDS;
}

//Returns the street names at the given intersection
std::vector<std::string> find_street_names_of_intersection(int intersection_id) {
    std::vector<std::string> streetSegments_Names;
    streetSegments_Names = StreetNamesofIntersections[intersection_id];
    return streetSegments_Names;
}


//Returns true if you can get from intersection_ids.first to intersection_ids.second
bool are_directly_connected(std::pair<int, int> intersection_ids) {
    //Corner Case
    if (intersection_ids.first == intersection_ids.second) return true;

    bool connectedIntersection = false;
    std::vector<int> AdjacentIntersections1 = find_adjacent_intersections(intersection_ids.first);
    for (int i = 0; i < AdjacentIntersections1.size(); i++) {
        if (AdjacentIntersections1[i] == intersection_ids.second) {
            connectedIntersection = true;
        }
    }
    return connectedIntersection;
}

//Returns all intersections reachable by traveling down one street segment
std::vector<int> find_adjacent_intersections(int intersection_id) {
    std::vector<int> adjacentIntersections;
    adjacentIntersections = AdjacentIntersections[intersection_id];
    return adjacentIntersections;
}

UI / UX

The team used EzGL, a graphics library, to visualize the map and display critical information for the users. Our main focus throughout this process was on designing a simple user interface (UI) and creating a pleasant user experience (UX).

Visual Features

  • Street names are displayed at proper zoom levels
  • Major roads are differentiated from minor roads
  • Street directions are shown (one-way or both ways)
  • Points of Interest including, but not limited to:
    • Parks
    • Beaches
    • Subway Stations
    • Hospitals
    • Banks
    • Restaurants

100% Customizable

we wanted to give as much control to the user as possible and to make every feature on the map Customizable. One of our ideas as a result of this was a mode that allows the user to fully customize the map color scheme as he or she wishes and filter specific content from the map.

Powerful Search Bar

We built a search bar with the following capabilities:

  • Allow users to find their destination using partial street names or mispelled names
  • Locate and highlight the intersection of two given streets
  • Find the shortest driving or walking path between two locations
  • Delete any past search-related data to improve the map's responsiveness
